# /cast — キャスティング

## 目的
シーンの登場キャラクターを抽出し、各キャラに「ペルソナ × 役」を割り当てる。
モデル（ローカルLLM / Opus）の指定もここで決定する。

## 入力
- **scene_detail**（必須）: シーン設定。登場人物・場所・時間・目的・感情の出発点と到着点
- **設計書**（任意）: Lシステムの/psyche, /dominator, /mechanics等の出力があれば参照

## 手順

### 1. キャラクター抽出
scene_detailから登場キャラクターを列挙する。
キャラクターごとに以下を整理：
- 名前・年齢・立場
- このシーンでの目的（何を得たいか、何を隠したいか）
- 感情の出発点
- 他キャラとの力関係

### 2. ペルソナ割当
各キャラクターに最適なペルソナを選ぶ。選定基準：

| キャラクターの特性 | 推奨ペルソナ |
|------------------|------------|
| 心理的に追い詰められている / 防衛的 | 晶（psyche） |
| 支配的 / 操作的 / 威圧的 | 橘（dominator） |
| 身体性が重要 / 無意識の反応 | 宮部（flesh） |
| 傍観者 / 情景の一部 / モノローグ中心 | 語り手（narrator） |

**意図的なミスキャスト**も有効な手段：
- 橘にウケ役を演じさせる → 支配者の視点から見た被支配の演技
- 晶にタチ役を演じさせる → 加虐の裏にある心理的脆弱性が滲む

### 3. モデル指定
役ごとにモデルを決定する。モデルの「癖」がキャラクターの質感になる：

| モデル | 出力傾向 | 向いている役 |
|--------|---------|------------|
| **Gemma 26B（local）** | 情念が強い。内面を身体で語る。説明的になりやすいが熱量がある | 受け手・内面の嵐を抱えたキャラ・感情を隠しきれない人物 |
| **Sonnet（API）** | 抑制的。行間・余白で語る。硬質で乾いた文体 | 支配者・威圧する側・言わないことで圧をかける人物 |
| **Opus（API）** | 推論が深い。心理の多層を正確に描く | 深い心理描写が核心のシーン。品質の最終基準 |

**キャラごとにモデルを分ける**のが最も効果的：
- 支配者（Sonnet）× 受け手（26B）→ 抑制と情念のコントラスト
- 逆配置で別テイクも有効：26Bの支配者は「教育者の熱」が出る、Sonnetの受け手は「感情を閉じ込めた少年」になる

### 4. キャスト表の出力

```markdown
## Cast Sheet: [シーン名]

| キャラクター | ペルソナ | モデル | 備考 |
|------------|---------|-------|------|
| ハインリヒ | 橘 | local | 義父の威圧、言葉の裏の支配意図 |
| アルベルト | 晶 | local | 防衛機制の裏の動揺、何を見抜かれたかの恐怖 |
| （語り手） | narrator | local | 書斎の空気、光の変化、沈黙の重さ |

### 演技順序
1. 語り手 → 場の導入
2. 橘 as ハインリヒ → 最初のアクション
3. 晶 as アルベルト → 応答
4. （以降リレー）

### テイク計画
- Take 1: 標準キャスト（上記）, temperature 0.8
- Take 2: 晶とnarrator入替（アルベルトの内面を語り手が俯瞰）, temperature 0.9
- Take 3: 全員Opus（比較用）
```

## ユーザーへの確認
キャスト表を提示したら、ユーザーに確認する：
- キャスティングの変更希望
- テイク計画の調整
- 追加したい実験的キャスト

ユーザーが「いい」「OK」「やって」と言えばそのまま/rehearsalへ進む。
