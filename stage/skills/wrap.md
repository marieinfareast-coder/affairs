# /wrap — 素材パッケージング

## 目的
/cutで選択・編集されたテイクを、執筆工程（/writeまたはKeiの智）に渡せる形式に整形する。
棄却素材・メタ議論・試行錯誤は含めない。クリーンな素材パッケージを作る。

## 入力
- /cutのカット表（採用パーツの一覧）
- 採用テイクのワークシート群
- scene_detail
- 設計書（あれば）

## 出力形式

```markdown
# Stage Output: [シーン名]

## Scene Detail
[scene_detailの転記。着地条件を明示]

## Design Notes（設計書がある場合）
[/psyche, /dominator, /mechanics等の設計書から、このシーンに関連する部分を抜粋]

## Selected Performance
[/cutで選択・編集された���フォーマンスを、シーンの時系列順に並べ直す]

### [場面1のタイトル]
[採用テイクの該当パート]

### [場面2のタイトル]
[採用テイクの該当パート]

...

## Director's Note
[監督からの執筆者への申し送り]
- このシーンで最も重要な瞬間
- 演技素材の中で特に活かしてほしい表現・フレーズ
- 演技素材にない補足（設計書にはあるが演技で拾えなかった要素）
- トーンの指定（テイクのどの温度感を基準にするか）
```

## 整形ルール

### 1. 時系列の再構成
/cutで混成した場合、パーツの時系列がずれることがある。
シーンの自然な流れに沿って並べ直す。

### 2. ペルソナ表記の除去
`[橘 as ハインリヒ]` → 不要。執筆者には誰の演技かではなく、キャラクターの行動として見せる。
ただし、内心表記 `（内心：...）` はそのまま残す（執筆者の心理描写の素材になる）。

### 3. 重複の除去
複数テイクから採用した場合、同じ情報が重複することがある。
自然な流れを壊さない範囲で重複を除去する。

### 4. 接続の補足
テイク間のパーツを繋いだとき、文脈の断絶があれば最小限の接続文を補足する。
ただし新しい演技を加えるのではなく、既存素材の橋渡しに留める。

## 渡し先

### Lシステムの/write
Lの設計フロー内で使う場合、このパッケージをLの/writeスキルに渡す。
/writeは設計書＋Stage素材を受け取り、別Agentで散文化する。

### Keiシステムの智
Keiのパイプライン内で使う場合��智（satoshi）への入力としてパッケージ��渡す。

### ユーザーへの直接提供
ユーザーが手元で編集する場合、パッケージをそのまま提示する。
