# /rehearsal — リレー演技

## 目的
/castで決定したキャスト表に基づき、共有ワークシート上で役者にリレー演技をさせる。
各役者は前の演技を読んでから自分の演技を行う。即興劇のように掛け合いの呼吸を生む。

## 入力
- **キャスト表**（/castの出力）
- **scene_detail**: シーン設定
- **設計書**（任意）: /psyche, /dominator等の出力

## ワークシート管理

### ファイル命名
```
workspace/[scene_id]_take[N].md
例: workspace/12-1_take1.md
```

### 初期化
ワークシートの冒頭にシーン設定とキャスト表を書き込む：

```markdown
# Worksheet: [シーン名] - Take [N]

## Scene
[scene_detailの内容をここに転記]

## Cast
- 橘 as ハインリヒ (local, temp=0.8)
- 晶 as アルベルト (local, temp=0.8)
- narrator as （語り手） (local, temp=0.7)

## Performance
```

## リレー演技の実行

### ローカルLLM呼び出し（標準）
```bash
python "./scripts/actor.py" \
  --persona "./personas/[persona].md" \
  --role "[キャラ名]（[補足情報]）" \
  --worksheet "./workspace/[file].md" \
  --temperature [temp]
```

呼び出し後、出力をワークシートに追記する：
```markdown

[ペルソナ名 as キャラ名]
（actor.pyの出力をここに貼り付け）
```

### Opus Agent呼び出し（重要シーン）
Agent toolで起動する場合のプロンプト構成：
```
あなたは演劇の役者です。

## ペルソナ
[personas/[file].mdの内容]

## 役
[キャラ名・背景・このシーンでの目的]

## ワークシート（ここまでの演技）
[現在のワークシート内容]

## 演技ルール
- パフォーマンス一体型で出力する
- セリフは「」、内心は（内心：...）、身体動作は地の文
- 前の演技を受けて自然に応答する
- メタコメント・挨拶・解説は禁止。演技のみ出力する
```

### リレー順序
/castで定めた演技順序に従って、1人ずつ呼び出す。
1ターン（1人の演技）の長さはシーン設定の規模から自然に決まる。
明示的な行数制限は設けないが、目安として：
- 短い応答（返事・反応）: 3-5行
- 通常の掛け合い: 5-10行
- 心理的に重要な場面: 10-15行

### リレーの終了判断
以下のいずれかで1テイクを終了する：
- scene_detailの着地条件（`==`）に到達した
- シーンの自然な区切り（退場・場面転換・沈黙の完成）に達した
- 監督（あなた）が「ここで切る」と判断した

## テイクのバリエーション

### 温度変化
- Take 1: temperature 0.7-0.8（安定した基準テイク）
- Take 2: temperature 0.9-1.0（冒険的な演技）
- Take 3: temperature 0.6（抑制的・繊細な演技）

### キャスト変更
- 同じキャラを別ペルソナに割り当てる
- ペルソナを入れ替える（意図的ミスキャスト）

### モデル変更
- ローカルLLM → Opus（同じキャストで深度を変える）

## 監督介入
リレー中に以下の判断を行う：
- **演技の軌道修正**: 役者の出力がシーンの方向性から外れた場合、次の役者への指示に補足を加える
- **追加ターン**: 掛け合いが盛り上がっている場合、予定外のターンを追加する
- **早期終了**: 着地条件に達した場合、残りの予定ターンをカットする
- **リテイク判断**: テイク全体が不調な場合、途中で破棄して新テイクを開始する

## 出力
完成したワークシートファイル（workspace/内）。
複数テイクがある場合は、全テイクを/cutに渡す。
