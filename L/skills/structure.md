# Skill: /structure — 章構成・ペース設計

## 目的
/plotの出力（プロット骨格）を、章単位の設計図に構造化する。
堕落の10段階ビートシートに沿った因果律を維持しつつ、「真綿で首を絞めるような遅滞」を設計する。

## 核心原則

### 狂気の構造化
プロットのギミック（点）を、ビートシートという因果律（線）に強制配置する。
ギミックの面白さを殺さず、むしろ絶望感を倍増させるための「伏線」と「回収」を設計する。

### 遅滞の設計
主人公の「理（ロジック）」が空回りし、逃げ場のないシステムに徐々に絡め取られていくプロセスを設計する。以下を文学的瑕疵（三流の展開）として排除する：
- 「早すぎる快楽」（Phase 5の地雷）
- 「あっさりとした屈服」（Phase 3の地雷）
- Phase飛ばし

## 出力フォーマット

### 冒頭宣言
- **文体ベースライン**: 作品全体のデフォルト文体を宣言する（文の長さ、形容詞の密度、視点、話法）。温度曲線の各章指示はこのベースラインからの差分として機能する
- **収束方向（一行）**: 作品末尾の着地方向性を一行で宣言する（収束タイプ、感情トーン、余韻の長さ）。結末の具体内容は書かない

### 章構成
各章を以下の形式で出力する：
```
第○章：Phase X（ビート名）
- この章で誰が何をし、主人公の心理がどう壊れるか
- 温度曲線上の位置づけ（ベースラインからの差分指示）
着地条件: [物理的・具体的に。抽象的総括は禁止]
```

### 温度曲線
章ごとの強度推移を設計する。質的指示を重視する（「追悼の手触り」「弔辞のような静けさ」等の指示は、数値パラメータより豊か）。

## 出力時の制約
- **小説本文禁止。** 地の文・台詞・描写は書かない
- 安易な解決やPhase飛ばしを禁止
- 構成の堅牢さを100点満点で自己評価する
- 着地条件は必ず物理的・具体的に記述する（「彼は成長した」のような抽象終了は許さない）
