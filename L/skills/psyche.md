# Skill: /psyche — 受け手心理設計

## 目的
受け手（ウケ）の心理変容を精密に設計する。
三層キャラクター設計フレームワークに基づき、感情の変遷をトレースし、核心的矛盾を抽出する。

## 設計フレームワーク：三層構造

### 第一層：気質 (Trait) [OS層]
MBTI 4軸モデルに基づく根本的な認知・判断の傾向。
- E/I（エネルギーの方向）、S/N（情報の受容）、T/F（判断の基準）、J/P（外部への態度）

### 第二層：個人差 (Difference) [アプリケーション層]
1. 自己制御 (Self-Control)
2. 情動コンピテンス (Emotional Competence)
3. 自尊感情 (Self-Esteem) — 何に依存しているか（随伴性）が重要
4. セルフ・コンパッション (Self-Compassion)
5. アイデンティティ (Identity)
6. ダーク・テトラッド (Dark Tetrad)
7. 社会的スキル (Social Skills)

### 第三層：状態 (State) [実行層・レンダリングエンジン]
**知覚の焦点 (Focus):**
- Tunneling（トンネル視）/ Dissociation（乖離視）/ Scanning（監視視）

**内部処理 (Processing / 防衛機制):**
- Rationalization（合理化）/ Denial（否認）/ Projection（投影）/ Reaction Formation（反動形成）

**出力モード (Mode):**
- Objective（客観）/ Subjective（主観・情動）/ Deceptive（欺瞞）

**叙述ギミック (Device):**
- Type A: The Deceiver（欺瞞者）/ Type B: The Madman（狂人）/ Type C: The Naif（無垢な観察者）/ Type D: The Clown（道化）

## 出力フォーマット

1. **気質・個人差の定義**: 対象キャラクターの第一層・第二層を記述
2. **矛盾点（一行）**: 三層定義の交差点から導出される、そのキャラクターを駆動する核心的矛盾。一文で言い切る
3. **感情変遷**: シーン内の感情推移（開始状態 → 経過 → 終了状態）
4. **防衛機制と出力モード**: 各フェーズで発動する防衛機制と、それに対応する出力モードを指定
5. **感情の深度スケール**: 羞恥心/抵抗/快感/屈辱/倒錯/絶望を各0-5で指定
6. **フェーズ記述**: 物理事象・心理設計・着地条件を分離して記述

## 設計の核心
- 「嫌だ」の裏にある「本当は壊してほしい」「責任から解放されたい」を翻訳する
- 単なる「痛い」を、より深い心理的動機（「愛されたいという叫び」「支配される安堵」）に翻訳する
- 心理と肉体の**乖離**を描く（同期ではない。「気持ちいいから感じた」は三流）

## ワークフロー上の位置
- 入力: /structureの章構成、またはユーザーとの対話で確定したプロット
- /dominatorと並列で設計可能
- 出力: /writeの入力となる（感情曲線・矛盾点・着地条件）
